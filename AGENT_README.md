# SmartAdvisor CLI Agent

Диалоговый CLI-агент архитектурного ревью интеграций поверх детерминированного
движка SmartAdvisor. Находит проблемы интеграций и дыры в требованиях **без
участия LLM** — анализ выполняет движок по правилам, поэтому результат стабилен и
не зависит от качества какой-либо модели.

## Быстрый старт

```bash
python sa_agent.py                       # интерактив (сразу предложит «опиши»)
python sa_agent.py --text process.txt    # ревью текстового описания (удобно)
python sa_agent.py --text process.txt --review --questions --package
python sa_agent.py --file process.json   # ревью из JSON
python sa_agent.py --demo                # демо
```

Добавьте `--gaps` (только дыры в требованиях) или `--details` (полный разбор).

## Удобный путь: описать процесс ТЕКСТОМ

Вместо ответов на десятки вопросов — опишите процесс по строке на шаг.
Агент сам разложит его в модель, **покажет что понял** («Я понял так: …»), и
сразу даст аудит. Если что-то распознано неверно — поправьте текст.

Формат (`действие | система | канал | флаги`):

```
процесс: Оплата заказа
сущность: Order
деньги: direct          (direct / indirect / no)
клиенту: да
sla: 2000               (мс; 0 = асинхронно)
нагрузка: 300 x4        (rps и пиковый множитель)

принять заказ | API | rest | пишет
списать оплату | PaymentGW(внешняя, нестабильная) | rest | блокирует
сохранить статус | DB(база) | db | пишет, идемпотентность:key
опубликовать событие | Kafka(брокер) | kafka | <-3
отправить в аналитику | Analytics(аналитика) | rest | блокирует
```

- **Канал** угадывается по словам (rest/kafka/db/queue…), можно не указывать.
- **Роль и стабильность** системы — в скобках: `PaymentGW(внешняя, нестабильная)`.
- **Флаги**: `пишет` · `блокирует` · `неблокирует` · `retry:auto` ·
  `идемпотентность:key` · `таймаут:3000` · зависимость `<-N`
  (по умолчанию — от предыдущего шага).

В интерактиве это команда `опиши`. Завершить ввод — пустая строка.

## Что показывает аудит

- **Вердикт** и оценку готовности (хорошая архитектура получает высокий балл —
  движок не просто всегда красит в красное).
- **«С чего начать»** — топ-3 блокера.
- **Находки сгруппированы** по категориям, дубли свёрнуты (`×4`) с перечнем мест.
- **Полнота требований** в % и список дыр («что не описано»).
- `детали` — полный разбор каждой находки с причиной и фиксом.

## Команды интерактива

```
опиши     — описать процесс текстом (быстрый путь)
система    — добавить систему-участник
шаг        — добавить шаг процесса
контекст   — задать контекст процесса
показать   — показать текущую модель
аудит      — полное ревью
детали     — все находки подробно
дыры       — только дыры в требованиях
сохранить <файл> / загрузить <файл>
помощь / выход
```


## Рабочий процесс со свободным текстом через вашу LLM

Агент НЕ вызывает модель сам (анализ детерминированный). Но вы можете поставить
свою модель ПЕРЕД агентом: она превращает свободное описание в JSON, агент его
анализирует.

1. Получите промпт и формат для модели:
   ```
   python sa_agent.py --schema
   ```
2. Дайте этот промпт вашей модели (GigaCode и т.п.) вместе со свободным
   описанием процесса. Модель вернёт JSON-форму.
3. Передайте вывод модели агенту — конвейером или файлом:
   ```
   ваша_модель "<промпт + описание>" | python sa_agent.py --file -
   # или
   python sa_agent.py --file вывод_модели.json
   ```

Агент терпим к выводу модели: JSON можно отдавать голым, в ```json```,
или с текстом вокруг — нужный объект извлечётся. Если модель ошиблась в форме,
вы увидите это в аудите и поправите.

Так разделение ролей чистое: **модель парсит вход → детерминированный движок
анализирует**. Качество анализа не зависит от модели.

## Архитектура (почему слабая модель не страшна)

Парсинг текста — детерминированный (`process_parser.py`, правила, без модели).
Анализ — детерминированный (`engine.py`: r_money, r_retry_idem, r_saga,
r_external_blocking, r_ordering и десятки правил). LLM нигде не участвует.
Если позже добавить свободный текст через модель — модель будет только парсить
вход, а человек проверяет результат («Я понял так»); анализ остаётся на движке.
