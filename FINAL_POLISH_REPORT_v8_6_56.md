# SmartAdvisor v8.6.56 — финальная полировка и жёсткая проверка

## Что было дополировано

1. **Визуальный сборщик учебного решения стал честным.**
   - Контроли больше не выбраны по умолчанию.
   - Слабый черновик, пользовательский выбор и эталон разделены.
   - Пользовательский payload теперь строится серверно только из реально выбранных контролей.
   - Пустой пользовательский выбор больше не превращается в почти эталонное решение.

2. **Добавлен серверный API для visual builder.**
   - Новый endpoint: `/api/learning/visual-payload`.
   - Режимы: `weak`, `selected`, `reference`.
   - Для `selected` учитывается список control id, выбранных пользователем в UI.

3. **Убрана гонка в UI.**
   - Проверка решения теперь дожидается сборки payload.
   - Кнопки “Собрать по моим выбранным контролям”, “Подставить эталон”, “Собрать слабый черновик” работают предсказуемо.

4. **Дополирован mobile UI.**
   - Увеличены интерактивные чекбоксы.
   - Поджаты заголовки, карточки и action-зоны.
   - Проверены desktop/tablet/mobile на отсутствие горизонтального overflow.

5. **Версия приведена к единому состоянию.**
   - UI: `8.6.56-polished-final`.
   - README обновлён.
   - Добавлены release notes `RELEASE_NOTES_v8_6_56.md`.

6. **Добавлены регрессионные тесты.**
   - Пустой пользовательский выбор не равен эталону.
   - Частичный выбор `outbox + timeouts` не подтягивает невыбранные DLQ/schemaVersion.
   - Эталонный режим по-прежнему даёт полноценный reference payload.

## Честный статус

Это уже не просто “MVP, который запускается”, а нормальная beta/product-pilot версия. Её можно использовать для портфолио, демонстрации, обучения, подготовки к собеседованиям и ограниченного платного пилота.

Для полноценного публичного SaaS всё ещё нужны продуктовые вещи вне текущего ядра: аккаунты, роли, биллинг, сохранение пользовательских проектов, журнал действий, телеметрия, админка и отдельная аналитика использования. Само ядро, учебный режим и UI после текущей полировки работают стабильно по пройденным проверкам.
