# No-text constructor update

Что сделано:

- Основной UI заменён на простой конструктор без ручного текста.
- Видимый путь: тип задачи → цепочка систем → поведение → проверка понимания → результат.
- В основном пути нет textarea, raw matrices, быстрых/глубоких/экспертных режимов.
- Пользователь выбирает только карточки, select, checkbox и готовые шаблоны.
- Старые технические секции сохранены только в закрытом блоке «Показать технические детали».
- Результат начинается с 4 блоков: схема, что обязательно сделать, главные риски, что отдать разработке.
- Полный markdown-отчёт скрыт за кнопкой/accordion «Открыть полный отчёт».
- Покрыты case_type: sync_rest, async_worker, event_kafka, enrichment_kafka, callback, dwh, legacy_file, status_aggregation, audit.
- Добавлены тесты `test_v60_no_text_constructor.py`.

Проверка:

- `python -m pytest -q test_v60_no_text_constructor.py` → 5 passed.
- Ручной программный прогон 9 case_type через Engine.generate/result_page прошёл.
- Проверено: в видимом основном UI до блока «Показать технические детали» нет `<textarea>`.

Ограничение:

- Полный старый набор UI-тестов содержит проверки прежнего интерфейса с режимами и старым wizard, поэтому часть старых UI-тестов нужно переписать под новый no-text flow.
