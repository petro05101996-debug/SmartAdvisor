# Проверка пользовательского пути v8.6.1

Проверялся путь обычного пользователя, который не понимает элементы и связи конструктора.

## Что изменено

- В простом режиме раздел называется «Сценарий процесса», а не «цепочка компонентов».
- Перед цепочкой добавлена инструкция «Как читать и строить цепочку».
- Участники процесса в простом режиме не заставляют редактировать карточки систем; они показаны как автосводка.
- Карточка шага объясняет:
  - что происходит;
  - после какого шага выполняется;
  - откуда берёт данные;
  - кто выполняет;
  - куда идёт результат.
- Сырые маршруты вида `source → system → target` убраны из простого восприятия.
- Добавлен блок «Продолжить цепочку простым действием»:
  - сохранить результат;
  - передать дальше;
  - дождаться статуса позже;
  - проверить данные;
  - сверить с другим источником;
  - сообщить другим системам;
  - отправить на ручной разбор.
- В простом режиме перетаскивание отключено. Перемещение оставлено в экспертном режиме.

## Проверка браузером

`python verify_ui_real_user_path_v860.py`

Результат: 32 ok, 0 fail.

## Вывод

Пользователь теперь строит не техническую цепочку из элементов и связей, а сценарий процесса из понятных действий. Связи и маршруты создаются автоматически. Экспертный режим нужен только для ручной корректировки.
