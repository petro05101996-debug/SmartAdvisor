# UX Advanced Simplification Report v4.9.1

## Цель
Сделать не только простой мастер, но и расширенный/сложный режим понятным для человека без глубокого опыта в интеграциях.

## Что изменено

1. Расширенный режим переименован и переосмыслен:
   - было: «Я аналитик, покажи всё»;
   - стало: «Расширенный режим: тоже пошагово».

2. Добавлен onboarding-блок для расширенного режима:
   - объясняет, что это не экзамен по архитектуре;
   - даёт простой порядок мышления: цель → системы → шаги → ошибки;
   - отдельно говорит, что неизвестное можно оставить как «не знаю».

3. В каждый сложный раздел добавлена человеческая инструкция:
   - зачем нужен блок;
   - как думать;
   - что можно оставить неопределённым;
   - какие решения потом примет инструмент.

4. Для сложных матриц добавлены подсказки рядом с полями:
   - systems_matrix;
   - process_steps;
   - error_matrix;
   - fields;
   - business_goal;
   - compromise_comment;
   - audit matrices.

5. Исправлена UX-ошибка простого мастера:
   - было два одинаковых поля «Название задачи» с одним id;
   - оставлено одно поле.

6. Добавлена кнопка возврата из расширенного режима в простой мастер.

## Проверка

Запущен полный regression suite:

```text
50 passed
```

## Вывод
Теперь сложный режим сохраняет всю мощность инструмента, но перестаёт выглядеть как архитектурная анкета. Пользователь видит пошаговую логику, человеческие объяснения и подсказки прямо рядом со сложными полями.
