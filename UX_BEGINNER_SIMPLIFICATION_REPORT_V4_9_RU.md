# UX/UI проверка и упрощение v4.9

## Цель
Сделать инструмент не только архитектурным rule-engine, а понятным инструктором по интеграциям для человека без глубокого знания Kafka, Outbox, Saga, CDC, DWH и других терминов.

## Что было проблемой
- Пользователь видел слишком много технических секций сразу.
- Даже при наличии beginner-блока ниже оставались матрицы, поля, wizard и подробные настройки.
- Результат начинался как большой технический Markdown, а не как простой ответ: что проектировать, что опасно, можно ли отдавать в разработку.
- Новичок мог потеряться между готовыми пресетами, wizard-шагами и расширенными полями.

## Что изменено

### 1. Простой режим по умолчанию
Теперь стартовый экран показывает только простой мастер:
- выбрать тип ситуации;
- указать название;
- перечислить системы;
- выбрать нагрузку;
- указать риск денег/регуляторики;
- выбрать стабильность внешних систем и примерное число шагов.

Сложные секции скрыты CSS-правилом до включения режима аналитика.

### 2. Отдельный режим аналитика
Кнопка `Я аналитик, покажи всё` включает полный режим:
- wizard;
- все секции;
- матрицы систем;
- матрицы шагов;
- error matrix;
- delivery/security/rollout/testing.

То есть новичок не пугается, а опытный системный аналитик не теряет мощность инструмента.

### 3. Верхний экран стал обучающим
Добавлен понятный путь:
1. Ситуация.
2. Риски.
3. Черновик.
4. Отчёт.

Это превращает приложение в инструктора, а не просто в анкету.

### 4. Результат теперь начинается с простого вывода
Вместо огромного Markdown сверху показывается:
- что проектировать;
- можно ли использовать вывод;
- что опасно;
- выжимка для новичка;
- альтернативы.

Полный Markdown спрятан в раскрываемый блок `Показать полный технический отчёт / ADR`.

### 5. В Markdown добавлен раздел 0
В каждый отчёт добавлен раздел:

`## 0. Финальное решение в 5 строк для новичка`

Он нужен, чтобы человек сразу понял:
- какой вариант проектировать;
- какие ключевые контроли нужны;
- что входит в MVP;
- что входит в production;
- какие blocking/high риски надо закрыть.

### 6. Проверка регрессий
Добавлены UX smoke-тесты:
- проверка, что beginner UI действительно простой по умолчанию;
- проверка, что результат содержит простую выжимку;
- проверка, что полный отчёт не потерян;
- проверка, что Markdown содержит раздел 0.

## Результат тестов

```text
48 passed in 0.37s
```

## Жёсткий вывод
После v4.9 инструмент стал ближе к настоящему инструктору:
- новичок может заполнить минимум и получить полезный черновик;
- системный аналитик может включить расширенный режим и точно описать сложный кейс;
- результат стал читаться сверху вниз: сначала решение простыми словами, потом технический ADR;
- архитектурная мощность не урезана.

Оставшаяся зона улучшения для следующих версий:
- добавить интерактивные подсказки “почему система выбрала этот паттерн” рядом с каждым результатом;
- сделать отдельный экран сравнения `MVP vs Production`;
- добавить визуальный конструктор цепочки вместо ручной матрицы `process_steps`;
- добавить импорт из простого текста: пользователь вставляет описание задачи, система пытается заполнить поля эвристиками без LLM.
