# V5.2 — радикальное упрощение UX и читаемого отчёта

## Что исправлено

- Обычный пользовательский режим больше не должен требовать ручного заполнения raw-матриц.
- Основной режим построен вокруг карточек сценариев, карточек систем и конструктора цепочки.
- Старые матрицы, DDL, длинный технический экспорт и raw appendix скрыты из обычного отчёта.
- Полный технический отчёт доступен только в expert mode.
- Простой отчёт теперь отвечает человеческим языком: что проектируем, что делать, почему, ограничения, риски, следующий шаг, тесты и краткий ADR.
- В обычном режиме добавлен hidden `report_detail=human`; expert mode переключает `report_detail=expert`.
- Матрицы `systems_matrix`, `process_steps`, `process_flow_matrix`, `target_integration_matrix`, `error_matrix` продолжают генерироваться автоматически из карточек/цепочки для совместимости движка.

## Что осталось намеренно

- Название проекта, короткая бизнес-цель и названия систем остаются ручным вводом: это единственное, что пользователь обычно должен назвать сам.
- Expert mode сохранён для системного аналитика/архитектора, которому нужны raw-матрицы, DDL и полный экспорт.
