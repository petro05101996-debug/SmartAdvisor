# V5.3 — Простота заполнения для всех режимов

Сделано:

- Основной сценарий и глубокий режим теперь используют один принцип: выбор вариантов и карточек, без ручных матриц.
- Глубокий режим отличается глубиной вопросов: управление цепочкой, fan-out/fan-in, циклы, recovery, ограничения source, инфраструктура, SLA, эксплуатационные риски.
- Raw matrices скрыты из пользовательского UI. Они остаются только как внутренний/export-формат для совместимости и тестов.
- Кнопка расширенного режима больше не ведёт пользователя в ручное заполнение systems_matrix/process_steps/contract_matrix/error_matrix.
- Отчёт в human-режиме не содержит Draft SQL DDL, raw-матриц и технической простыни.
- Технический markdown/ADR показывается свёрнутым и содержит human markdown, а не старый сырой отчёт.

Проверено:

- pytest -q --ignore=tests/ui → 174 passed.
- python -m compileall → OK.
- HTTP GET / → 200.
- HTTP POST /generate → 200.
- В human отчёте нет `Draft SQL`, `create table`, `Введённые матрицы`, `Матрица систем`.
