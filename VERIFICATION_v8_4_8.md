# VERIFICATION v8.4.8 — auto stack selection

## Цель правки

Пользователь больше не обязан выбирать REST/Kafka/SOAP/Batch/CDC как основной путь. UI оставляет текущий конструктор и экспертный режим, но добавляет автоподбор способа взаимодействия по смыслу шага.

## Что изменено

- Добавлен автоподбор канала/стека для шага: `inferChannelForStep` + `applyAutoStackForStepAt`.
- При перестановке, drag-and-drop, изменении смысла шага, источника/исполнителя/получателя и перед отправкой payload стек пересчитывается автоматически, если эксперт явно не включил ручное переопределение.
- В карточке шага отображается: «Стек подбирается автоматически: REST/Kafka/БД/Batch/CDC» и причина выбора.
- Ручной выбор канала спрятан в раскрываемый экспертный блок «Переопределить стек вручную».
- Добавлена кнопка «вернуть автоподбор».
- Существующее ядро анализа и формат payload не изменены.
