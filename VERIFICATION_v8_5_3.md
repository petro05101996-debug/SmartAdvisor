# Проверка v8.5.3 — русские отчёты и понятные описания

## Что изменено

1. В отчётах добавлен слой русификации пользовательских текстов.
   Внутренние коды движка не менялись, чтобы не сломать правила анализа.

2. Технические названия стеков сохранены только там, где это реальные имена технологий:
   Kafka, RabbitMQ, Redis, REST, SOAP, gRPC, CDC, SFTP.
   Рядом с ними добавлены русские пояснения: поток событий, очередь задач, кэш, обратный вызов, файловый обмен и т.д.

3. Лишние англицизмы в отчётах раскрываются русскими словами:
   retry → повторная попытка;
   DLQ → очередь ошибочных сообщений;
   replay → повторная обработка;
   callback → обратный вызов;
   webhook → входящий веб-вызов;
   fallback → запасной сценарий;
   rollback → откат;
   cutover → переключение;
   runbook → инструкция разбора;
   event envelope → обёртка события;
   payload → тело сообщения;
   rate limit → лимит запросов;
   watermark → контрольная отметка загрузки.

4. В основном UI стек теперь отображается с пояснениями:
   «RabbitMQ — очередь задач», «Redis — кэш для быстрого чтения»,
   «CDC — передача изменений из БД», «Batch — пакетная обработка по расписанию».

5. Разделы отчёта переименованы на русский язык:
   «Контрольные проверки готовности к промышленному запуску»,
   «Готовность к началу разработки»,
   «Готовность к выпуску»,
   «План внедрения и безопасного вывода в промышленную эксплуатацию».
