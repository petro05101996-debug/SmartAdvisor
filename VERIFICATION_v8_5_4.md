# Проверка v8.5.4 — отчёт с объяснением решений

Цель доработки: отчёт должен быть понятным человеку. Если выбрана технология или архитектурный паттерн, отчёт обязан объяснить:

- почему выбрано именно это решение;
- почему не выбран другой вариант;
- какие обязательные условия нужны для безопасной реализации;
- какие риски останутся, если решение не применять.

## Что изменено

1. Добавлен раздел отчёта `Почему выбраны технологии и способы взаимодействия`.
2. Для каждого шага выводится таблица:
   - шаг и маршрут;
   - технология / способ взаимодействия;
   - почему выбрано;
   - почему не другой вариант;
   - обязательные условия.
3. В сценарной основе для разработки добавлено объяснение выбранного способа взаимодействия.
4. В вариантах архитектурного решения добавлено:
   - почему предлагается именно такой вариант;
   - почему это не универсальный ответ.
5. В разделе рекомендуемых паттернов добавлено:
   - почему паттерн предлагается;
   - почему нельзя просто не делать;
   - какие контроли обязательны.
6. Внутренняя модель шага теперь сохраняет `source_system`, `target_system`, `stack_reason`, чтобы отчёт мог объяснять не только технологию, но и маршрут шага.
