## Проверенный пользовательский путь

1. Пользователь выбирает смысл процесса в мастере: старт, действие, момент результата, действие с результатом, масштаб.
2. Конструктор собирает черновую цепочку без выбора REST/Kafka/RabbitMQ/Redis/SOAP.
3. Пользователь добавляет смысловые уточнения: быстрый доступ на чтение, история событий, аналитика, фоновая обработка, документы, безопасность и т.д.
4. Пользователь нажимает «Определить стек по процессу».
5. Система предлагает технологии и объясняет, почему выбрано именно так.
6. В экспертном режиме пользователь может поправить технологию вручную; доступен весь каталог из 55 вариантов.
7. При перемещении шагов пересчитываются маршрут, зависимости и автоподбор.
8. Отчёт содержит разделы «почему выбрана технология», «почему не другой вариант», «что обязательно проверить».

## Ограничения

Это production-polish для MVP-конструктора. Перед реальным промышленным использованием всё равно нужны внешние нефункциональные требования конкретной организации: SSO, роли, аудит действий пользователей, хранение историй запусков, резервное копирование, мониторинг контейнера, секреты через окружение/хранилище секретов и security-review.
